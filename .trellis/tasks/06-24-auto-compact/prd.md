# Auto-compact history when context nears limit

## Goal

Automatically summarize conversation history when the input token count
nears the model's context window. Replaces the full history with a
compact summary plus the most recent few messages, letting long-running
sessions continue past the context limit without manual intervention.

## Background

pyccode's three-layer context management (per-result size, per-message
size, per-turn count) keeps individual tool outputs bounded, but the
conversation history itself grows unboundedly across turns. After many
turns, the API request size approaches the model's context window and
eventually fails with `prompt_too_long`.

Claude Code solves this with auto-compact: before each API call, if the
previous response's `input_tokens` exceeds a threshold, summarize the
history and replace it with `[boundary, summary, recent_N]`. The
summary is generated by a separate LLM call with no tools, using a
9-section prompt template.

This task ports that pattern to pyccode as an MVP.

## Requirements

### Functional

- New function `maybeAutoCompact(history) -> bool`:
  - Reads module-level `_last_input_tokens` (updated by `chat()` after
    each API response).
  - If `_last_input_tokens <= AUTOCOMPACT_THRESHOLD`, returns `False`
    without modifying history.
  - If `len(history) < AUTOCOMPACT_KEEP_RECENT + 2`, returns `False`
    (not enough history to be worth compacting).
  - Otherwise: calls the model with a 9-section summary prompt and the
    full history as input; replaces history in place with
    `[boundary_msg, summary_msg, *recent_N_msgs]`.
  - Returns `True` if compacted.
- The summary LLM call:
  - Uses the same model as the main agent (`MODEL_NAME` env).
  - Sends no tools (text-only response enforced by omission).
  - Uses `AUTOCOMPACT_MAX_OUTPUT_TOKENS` (16384) as the output cap.
  - Uses a fixed system prompt: `"You are a helpful AI assistant tasked
    with summarizing conversations."`
- Boundary message: user role, content `[compact_boundary]`. Appended
  to history and transcript via `_history_append`.
- Summary message: user role, content is a prefix + the LLM-generated
  summary. Prefix tells the model:
  - "This session is being continued from a previous conversation that
    ran out of context."
  - "Do not recap or ask the user what to do next — continue the work
    from where it left off."
  - "If you need specific details from before compaction, read the full
    transcript at: `<TRANSCRIPT_PATH>`."
  Appended to history and transcript via `_history_append`.
- Recent N messages (last `AUTOCOMPACT_KEEP_RECENT = 4` entries of
  history) are preserved as-is, **not** re-appended to transcript
  (they're already there from when they were first written).
- `chat()` updates `_last_input_tokens` after every API response and
  calls `maybeAutoCompact(history)` at the top of the next loop
  iteration.
- First turn (no previous response): `_last_input_tokens = 0`, no
  compact attempt.
- `handle_subagent()` does NOT add auto-compact (subagent tasks are
  short by design).
- On any failure during compact (LLM error, empty response, network):
  print a yellow notice to stderr, return `False` without modifying
  history. The main loop continues normally; next turn will retry.

### Non-functional

- Six new module constants for thresholds and limits.
- One new module-level `_last_input_tokens` state variable.
- Two new functions: `maybeAutoCompact(history)` and a private
  `_callCompactLLM(history) -> str` helper.
- Summary prompt template stored as a module-level string constant
  `AUTOCOMPACT_PROMPT`.
- No new external dependencies.
- No real tokenizer — uses `response.usage.input_tokens` from the
  previous API call.

## Acceptance Criteria

- [ ] A history that produced `_last_input_tokens > AUTOCOMPACT_THRESHOLD`
      triggers a compact at the top of the next turn.
- [ ] After compact, history is `[boundary_msg, summary_msg, *last_4_msgs]`
      — full prior history is gone.
- [ ] The boundary message appears in transcript as a regular user
      entry with content `[compact_boundary]`.
- [ ] The summary message appears in transcript with the prefix
      described above plus the LLM-generated summary text.
- [ ] Recent 4 messages are NOT re-written to transcript (they appear
      exactly once, at their original positions).
- [ ] A history below the threshold is returned unchanged.
- [ ] A history with fewer than `AUTOCOMPACT_KEEP_RECENT + 2` entries
      is returned unchanged even if `_last_input_tokens` is high.
- [ ] A failed compact LLM call (e.g. simulated network error) returns
      `False`, prints a yellow stderr notice, and leaves history
      untouched.
- [ ] `_last_input_tokens` is correctly updated after every successful
      API response in `chat()`.
- [ ] The chat loop continues normally after a compact (next API call
      sees the smaller history).

## Out of Scope

- Predictive auto-compact (triggering based on estimated next-turn size).
- Session Memory compact (Claude Code's first-stage strategy).
- Failure counter / disable mechanism (Claude Code disables after 3
  consecutive failures).
- Subagent auto-compact.
- Real tokenizer integration.
- Configurable thresholds via env vars (constants only for MVP).
- Re-injection of skill metadata or other first-message adornments
  after compact.
- Compact-specific tool restrictions preamble (Claude Code's
  `NO_TOOLS_PREAMBLE`); we achieve text-only by simply not passing
  tools to the summary call.
- Resume / load logic that reads the boundary marker to rebuild state.
- Multiple compact strategies / pluggable compact function.
